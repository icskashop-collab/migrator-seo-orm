★ Главный козырь — кейс migrator.me

  • Index bloat: 581 стр. в индексе при 31 реальной (раздут в 18×)
  • Tilda-проблемы: HTML до 1,1 МБ, текст <2% кода, 0 hreflang, 0 JSON-LD
  • 7 групп каннибализации (3 кейса под «EB-1A маркетолог»)
  • Слабые ссылки: TF 5, 47 доменов, 90% RU-доноров при таргете на США
  • Перелинковка: money-страницы по 2 входящих, анкор «читать полностью» ×63
  • Решение: ICE-приоритет → noindex мусора, hreflang, H1, pillar+cluster, разметка

Стратегия с нуля

  • Аудит + бенчмарк конкурентов
  • Семантика → кластеры по интентам
  • Структура pillar + cluster
  • Технический фундамент (CWV, индексация, hreflang, schema)
  • Контент + линкбилдинг = процессы
  • Замер по KPI, итерации
  • Суть сеньора — приоритизация (ICE/RICE)

pillar + cluster

  • Pillar — money-страница услуги
  • Cluster — статьи вокруг темы, ссылаются на pillar
  • Даёт: вес (PageRank) + релевантность (авторитетность) + воронку
  • На полезное ссылаются извне → растёт весь домен

noindex / canonical / robots / 301

  • robots.txt — запрет краула (не удаляет из индекса)
  • noindex — запрет индексации (не закрывай в robots!)
  • canonical — «вот основная версия», рекомендация
  • 301 — переезд, передаёт вес, склеивает

Core Web Vitals

  • LCP — загрузка контента <2,5 с
  • INP — отзывчивость <200 мс (заменил FID)
  • CLS — стабильность вёрстки <0,1
  • Смотреть полевые данные (CrUX/GSC), не только лабораторные
  • Причины: тяжёлые картинки, блокирующий JS, медленный сервер

Crawl budget

  • Проблема для крупных сайтов и при мусоре в индексе
  • Враги: параметры, фасеты, пагинация, тонкие дубли
  • Лечить: noindex/canonical, чистый sitemap, убрать редирект-цепочки, облегчить HTML

hreflang

  • Связывает языковые версии → нет каннибализации
  • Двусторонние ссылки + обязательный x-default
  • В head / HTTP / sitemap
  • Без него поисковик сам выбирает версию

JS / SPA рендеринг

  • Google рендерит в 2 волны → задержка
  • Яндекс с JS работает плохо
  • Решение: SSR / пререндеринг
  • Симптом: контент есть, но Screaming Frog не видит

Микроразметка (schema)

  • Organization + sameAs — бренд-панель
  • FAQPage — расширенный сниппет
  • Review/AggregateRating — звёзды, ↑CTR
  • Article, VideoObject, BreadcrumbList
  • JSON-LD, проверка в Rich Results Test

Каннибализация

  • Несколько страниц на один запрос → обе проседают
  • Ищу: GSC (запрос → разные URL), пересечение title/интента
  • Лечу: 301/canonical (дубли) ИЛИ развести по под-интентам (разный контент) ИЛИ hreflang (языки)

E-E-A-T (важно для YMYL)

  • Experience, Expertise, Authoritativeness, Trust
  • Авторы с био, About, лицензии с рабочими ссылками
  • Отзывы (Review), упоминания бренда, актуальные даты
  • Пример: битые ссылки лицензий = удар по доверию

Ссылки / off-page

  • Качество > количество: уникальные домены, траст, релевантность
  • Методы: linkable assets, гостевые, digital PR, посевы
  • Темп плавный, 5–10 доменов/мес, без всплесков
  • Токсичные: disavow только при реальном вреде/фильтре

⚠ Трафик упал на 30% — план

  • Реальный спад или сбой аналитики?
  • Что упало: сайт / раздел / страна / устройство?
  • Когда: апдейт / релиз / миграция / сезон?
  • Тех: noindex, robots, сервер, редиректы после деплоя
  • Позиции (алгоритм) vs CTR (сниппеты, AI Overviews)?
  • → гипотеза → фикс → замер

KPI и отчётность

  • Веду к деньгам: трафик → лиды → выручка
  • + ассистированные конверсии (SEO в начале воронки)
  • Опережающие: индексация, видимость, рост запросов, CTR
  • Не «позиции ради позиций»

AI / GEO / тренды 2026

  • Zero-click растёт, инфо-CTR падает
  • GEO/AEO — оптимизация под цитаты в AI
  • Помогает: структура, FAQ, schema, E-E-A-T, бренд-упоминания
  • AI-контент — инструмент + экспертная правка (Helpful Content)
  • Google vs Яндекс: ссылки/E-E-A-T vs поведенческие/ПФ

★ STAR — каркас кейса

S — Tilda-сайт, раздутый индекс, слабая структура
T — навести тех-порядок + заложить рост контентом
A — аудит, ICE, pillar+cluster, hreflang, schema, перелинковка
R — вставь свои реальные цифры! (трафик +X%, лиды +Y%)
