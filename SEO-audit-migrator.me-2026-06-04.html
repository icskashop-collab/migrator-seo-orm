1. Executive Summary

4.3 / 10

 Базовая гигиена основных страниц в порядке (200-коды, уникальные Title/Description, самоканоникалы, корректный robots.txt). Но сайт системно теряет видимость из-за раздутого индекса рекламными лендингами, полного отсутствия hreflang при 4 языковых версиях, отсутствия микроразметки и сверхтяжёлых страниц. Семь ключевых выводов

Все проверенные — индексируемые, без noindex, с одинаковым Title «Иммиграция в США!» и одинаковым Description. Это классический index bloat + массовые дубли, размывающие краулинговый бюджет и качество домена.
  • hreflang отсутствует полностью — при наличии EN/RU/ES/FR версий главной, страниц виз и asylum-страниц (/political*). Поисковики не понимают связь языковых версий; в выдаче конкурируют между собой и показываются не той аудитории. Для международного проекта это критично.

ИКС 130 (+10), по Яндексу есть ТОП-1 по запросам «сша внж» (~816 переходов/мес по оценке xtool), «профессии востребованные в сша» и др., ~116 переходов/сутки. 3. Детальный разбор

3.1 Техническое SEO

Что хорошо Все 31 краулящихся URL отдают 200, редиректных цепочек и циклов внутри сайта нет, 4xx/5xx — 0 (Screaming Frog). HTTPS работает, www → non-www редирект на месте (проверено 04.06.2026). robots.txt корректен: закрыты служебные пути Tilda (/tilda/form*, /tilda/rec* и др.), есть Clean-param: etext&gbraid&brid для рекламных меток и ссылки на оба sitemap. Канониклы — самоссылающиеся на 100% страниц, конфликтов нет. ЧПУ у кейсов читаемые (/eb1a-sergey-media-editor). Санкций Яндекса нет, вероятность фильтра — 0% (xtool).
Что плохо
  • Index bloat. sitemap.xml содержит 180 URL, из них ~150 — не связанные перелинковкой promo-лендинги, thank-you и формы. Проверка живых страниц (/varlamov31, /kats18, /varlamov25, /thanks): все 200, индексируемые, без noindex, с self-canonical. В индексе Яндекса 581 страница, Google ~540 — поисковики тратят бюджет на мусор, а «качество домена» считается по сотне одинаковых страниц.
  • hreflang отсутствует на всех страницах (0 упоминаний в 38 файлах бэкапа). Языковые версии связаны только видимым переключателем.
  • Вес страниц. Средний HTML — 478 КБ (без ресурсов!), 20 страниц >500 КБ, /otzyvy — 1 138 КБ. Среднее время ответа 0,99 с, у 15 страниц >1 с, максимум 1,93 с (/visao1). Это до загрузки CSS/JS/изображений Tilda — LCP и INP на мобильных почти гарантированно вне «зелёной зоны». Полевые данные CWV (CrUX) недоступны ни в одном из источников — требуется проверка в PageSpeed Insights/Search Console; отмечено как пробел данных.
  • HTTP/1.1 на всех URL (колонка HTTP Version краулинга) — десятки ресурсов Tilda CDN грузятся без мультиплексирования.
  • Sitemap не соответствует реальной структуре: в нём есть технические URL (/page115964576.html, /page116387006.html, /page116756016.html, /page117321176.html, /preview), которых не должно быть в принципе.
  • Глубина и доступность контента: кейсы лежат на глубине 3 клика и подгружаются JS-лентой Tilda — в статичном HTML страницы /news ссылок на кейсы нет (проверено по бэкапу). Краулеры без рендеринга JS их не увидят.
Как исправить 1) В Tilda массово закрыть от индексации все лендинги под трафик блогеров, thank-you, формы, опросы; пересобрать sitemap только из контентных URL. 2) Внедрить hreflang-кластеры (см. 3.4). 3) Аудит блоков Tilda на самых тяжёлых страницах: убрать дублирующиеся попапы (на главной форма-анкета продублирована дважды), включить lazy-load видео через фасады, конвертировать изображения в WebP (через настройки Tilda «оптимизация изображений»). 4) Запросить у Tilda поддержку HTTP/2 / проверить текущую отдачу с CDN. 5) Снять CWV в Search Console и PSI как базовую точку.

3.2 On-page SEO

Что хорошо На 31 основном URL: 0 пустых Title, 0 пустых Description, 0 дублей Title. Метатеги кейсов написаны осмысленно, с брендом и сутью кейса. Главная: Title «Immigration to the USA legally | Migrator» (41 зн.), внятный Description с CTA. Контент кейсов объёмный — 1 271–1 918 слов. Читабельность нормальная (Flesch 57–64 для EN-страниц).
Что плохо
  • H1 нет на 19 из 31 страниц (все 16 кейсов, /news, /otzyvy, /pp); H2 нет ни на одной — заголовки свёрстаны div-ами. xtool: оптимизация H1–H6 главной = 0%, в заголовках отсутствуют слова «immigration, usa, legally» из Title.
  • Длина метатегов: 20/31 Title длиннее 60 знаков (до 109 зн. — кейс архитектора), 13/31 Description длиннее 160 знаков (до 199 зн.) — обрезаются в выдаче.
  • Полный дубль Description у пары кейсов eb1a-ilya-automotive-ceo и eb1a-vladimir-tech-entrepreneur (описание Vladimir скопировано в кейс Ilya, текст не соответствует содержанию).
  • Дубли на лендингах: все проверенные blogger-лендинги имеют одинаковый Title «Иммиграция в США!» и одинаковый Description — десятки страниц-клонов (вне зоны видимости Screaming Frog, т.к. орфаны).
  • Тонкий контент: /news — 343 слова, /otzyvy — 495 слов. Доля текста <2% HTML на 16 страницах (балласт кода).
  • Alt-атрибуты: на контентных страницах большинство «настоящих» img имеют alt, но на главной 6 из 6, на /tg 8 из 16, на /esp и /fr по 6 из 11 изображений без alt (бэкап). Значительная часть графики — фоновые div Tilda, вообще невидимые для image-поиска.
  • xtool, релевантность главной по фразе из Title: текст 38%, ссылки 13%, текст+ссылки 33% — ключи почти не поддержаны анкорами внутренних ссылок.
  • Устаревший meta keywords на 3 страницах.
Как исправить 1) Пройтись по всем страницам в Tilda и назначить семантические H1/H2/H3. 2) Переписать 20 длинных Title (ключ в первые 50 зн.) и 13 Description; устранить дубль у кейса Ilya CEO. 3) Расширить /news (хаб с анонсами + регулярные статьи) и /otzyvy (текстовые версии видеоотзывов — это и контент, и доступность). 4) Заполнить alt у всех значимых изображений (фото юристов — «Olga Prygoda — US immigration attorney…»). 5) Удалить meta keywords.

3.3 Микроразметка и структурированные данные

Что хорошо Open Graph присутствует на всех основных страницах (og:title, og:description, og:image, og:type, og:url). Есть favicon (light/dark), apple-touch-icon, верификации Яндекс/Google.
Что плохо JSON-LD — 0 блоков на всех 38 страницах бэкапа и проверенных живых страницах. Нет Organization, Service, FAQPage, Review/AggregateRating, Article, BreadcrumbList. Twitter Cards — 0. og:image главной — файл «Screenshot_-_2025-06.png» (скриншот вместо брендированной обложки). При 16 детальных кейсах и видеоотзывах сайт не размечает ничего — поисковики и LLM получают только сырой текст в тяжёлом коде.
Как исправить Через «HTML-код в head» в Tilda добавить: на все страницы — Organization (name, url, logo, sameAs: YouTube/Instagram/Telegram/TikTok, contactPoint с телефоном +1-323-283-87-76, address); на страницы виз — Service + FAQPage (см. 3.6); на кейсы — Article с datePublished/author; на /otzyvy — VideoObject для видеоотзывов; везде — BreadcrumbList. Заменить og:image, добавить twitter:card. Валидация: Rich Results Test + Яндекс Вебмастер.

3.4 Структура сайта, hreflang и внутренняя перелинковка

Что хорошо Ядро структуры логично: главная (74 inlinks) → страницы виз (EB, O-1, по языкам) → кейсы. Главная связана со всеми хабами, глубина основных коммерческих страниц — 1 клик. ЧПУ кейсов человекочитаемые. /pp (privacy) корректно прилинкована отовсюду (62 inlinks).
Что плохо
  • hreflang-кластеры не размечены. Фактические кластеры: главные (/ EN, /tg RU, /esp ES, /fr FR), EB-визы (/eb, /ebru, /ebesp, /ebfr), O-1 (/visao1, /visao1ru, /visao1esp, /visao1fr), asylum (/political, /politicalru, /politicalesp, /politicalfr). В sitemap есть даже /aimigratorportu (PT) — языков больше, чем кажется.
  • 16 кейсов и /visao1* получают всего по 3 внутренних ссылки, кейсы — на глубине 3 и доступны только через JS-ленту. Самый убедительный контент сайта почти не получает внутреннего веса.
  • Орфаны: весь блок /political* (4 языка услуги «политическое убежище»), /whole_world, /migrator_reviews и ~150 promo-URL существуют вне перелинковки. Screaming Frog их не нашёл — это и есть определение orphan pages.
  • Анкоры: ссылки на визы с главной — «More →»; xtool фиксирует релевантность ссылок 13%, в анкорах отсутствуют «immigration, usa, legally». Несколько визуально похожих ссылок «/» с пустым анкором.
  • Языковое меню непоследовательно: с /tg нет прямой ссылки на /esp-кейсы и наоборот; RU-кейсы не имеют языковых аналогов вовсе.
Как исправить 1) hreflang по кластерам + x-default на EN. 2) Статичные блоки «Наши кейсы» на главной и страницах виз (3–4 ссылки с информативными анкорами «Кейс EB-1A: журналист и главред», «EB-2 NIW для IT-разработчика»). 3) Перекрёстные ссылки кейс → страница соответствующей визы (сейчас кейсы ссылаются в основном на /ebru) и виза → кейсы. 4) Вернуть /political* в меню или закрыть от индексации. 5) Заменить «More →» на анкоры с ключами («EB visa for professionals», «O-1 talent visa»).

3.5 Контент, коммерческие факторы и E-E-A-T

Что хорошо Сильная кейс-база: 16 подробных разборов (1,3–1,9 тыс. слов) с историей, стратегией, сроками, RFE/NOID — редкий для ниши уровень конкретики. Представлены юристы с именами, фото, опытом, номерами лицензий (WSBA #57879), членством в AILA. Есть видеоотзывы, цены «от $15 000», гарантия возврата в договоре, юридический адрес (TX), Delaware File Number 2604257 со ссылкой на официальный реестр, телефон, e-mail. Честные дисклеймеры в футере («не юрфирма», расшифровка «95%» и «2143 клиента»). Работают SERM-страницы (/migrator_scam, /migrator_scam_eng). Метрика и пиксели стоят (Яндекс.Метрика, Facebook Pixel).

3.6 GEO / AEO / видимость в LLM и AI-поиске

Что хорошо Сырьё для AI-видимости отличное: уникальные первичные данные (16 реальных кейсов с цифрами, сроками, типами петиций, исходами RFE/NOID/отказов), факты о ценах, статистика одобрений. Тематика «O-1/EB-1A/EB-2 NIW для русскоязычных» — узкая ниша, где LLM мало кого могут цитировать.
Что плохо
  • Контент технически почти нечитаем для retrieval: текст <2% HTML на половине страниц, заголовков H1–H6 нет, FAQ-блоков с разметкой нет, JSON-LD нет, lang не указан. LLM-краулеры (GPTBot, ClaudeBot, PerplexityBot) парсят сырой HTML без выполнения JS — лента кейсов на /news для них пуста.
  • Нет страниц, отвечающих на вопросы прямо: «Сколько стоит виза O-1?», «Сколько рассматривается EB-1A?», «O-1 vs EB-1A» — ответы разбросаны по лендингам в виде картинок и стилизованных блоков.
  • robots.txt не определяет политику для AI-ботов (ни разрешений, ни запретов — по умолчанию доступ есть, но и приоритетов нет).
  • Бренд почти не имеет независимых упоминаний (47 ссылающихся доменов, 0 EDU/GOV) — LLM нечем подтверждать сущность «Migrator».
Как исправить 1) На каждой странице визы — FAQ-секция из 5–8 вопросов с краткими фактологичными ответами (40–60 слов, цифры, сроки) + разметка FAQPage. 2) Переверстать ключевые факты из графики в текст: таблицы «стоимость / сроки / требования» обычным HTML — это то, что цитируют AI-обзоры. 3) Семантические заголовки (см. 3.2) — основа chunk-инга в RAG-системах. 4) Страница «O-1 vs EB-1A vs EB-2 NIW» со сравнительной таблицей — типовой AI-запрос ниши. 5) Наращивать сущность бренда: Wikipedia-grade источники недостижимы, но профильные каталоги, юр. директории, подкасты и YouTube-транскрипты дают LLM подтверждение существования компании. 6) Транскрибировать видеоотзывы в текст на /otzyvy.

3.7 Скорость и мобильная адаптивность

Что хорошо Tilda отдаёт адаптивную вёрстку, viewport настроен, шрифт Inter с display=swap, dns-prefetch/preconnect для CDN есть. Часть CSS грузится с media="print" → onload (отложенная загрузка).
Что плохо Вес HTML (до 1,1 МБ) и сотни DOM-узлов Tilda + 25–32 изображения на кейсах/отзывах + видеоembed-ы. Ответ сервера до 1,9 с. Лабораторных и полевых CWV-данных ни один источник не содержит — LCP/CLS/INP не подтверждены измерениями, это пробел, который надо закрыть через Search Console.
Как исправить Снять PSI/CrUX базу; включить оптимизацию изображений Tilda (WebP, lazy); фасады YouTube; ревизия блоков «Zero Block» с дублированным контентом (4 повтора бегунка на главной); удалить вторую копию анкеты-попапа.
